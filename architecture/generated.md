## Streaming Protocol (`StreamExecuteTool`)

`StreamExecuteTool` uses server-side streaming. Each message is a `ToolStreamEvent`
containing a `request_id` and one of four event variants:

### Event Types

| Variant | Type | Fields | Description |
|---------|------|--------|-------------|
| `Progress` | `ProgressEvent` | `message: String`, `percent: u32` | Execution progress update (0-100). |
| `Finding` | `FindingEvent` | `finding: Option<Finding>` | Emitted as findings are discovered (incremental). |
| `Complete` | `CompleteEvent` | `response: Option<ExecuteToolResponse>` | Terminal event with final results. |
| `Error` | `ErrorEvent` | `message: String`, `code: String` | Terminal event indicating failure. |

### Streaming Flow

```
Client                     Server
  |--- ExecuteToolRequest -->|
  |<-- ProgressEvent --------|  (0%)
  |<-- FindingEvent ---------|  (finding #1)
  |<-- ProgressEvent --------|  (45%)
  |<-- FindingEvent ---------|  (finding #2)
  |<-- ProgressEvent --------|  (100%)
  |<-- CompleteEvent --------|  (terminal)
```

The stream ends after a `CompleteEvent` or `ErrorEvent`. Clients should not expect
further messages after either terminal event.
